================================================================================
📋 ROOT CAUSE IDENTIFIED
================================================================================

1. Missing HTML Selectors for Wolipop Subdomain
   - Wolipop uses .itp_bodycontent class
   - Only news.detik.com selectors were implemented
   - Content extraction returned None for wolipop articles

2. Unsafe None Handling in API Endpoint
   - len(detail_data.get('content', '')) crashes if content is None
   - .get() with default doesn't help if key exists with None value
   - TypeError caused HTTP 500 error

================================================================================
✅ SOLUTION IMPLEMENTED
================================================================================

FILE 1: src/core/article_detail_scraper.py
- Added .itp_bodycontent selector (Wolipop, Food, etc)
- Added .detail__body selector (General detik)
- Reordered selectors by priority
- Now supports ALL detik subdomains

FILE 2: src/api/main.py
- Added safe content handling: content = detail_data.get('content') or ''
- No more TypeError on len() operation
- Graceful handling of missing content
