================================================================================
                              THE PROBLEM
================================================================================

Error Message: ModuleNotFoundError: No module named 'utils.db_helper'
Real Issue: ModuleNotFoundError: No module named 'utils.logger'

Import Chain:
  Dashboard → utils/db_helper ✓
    └→ storage/database (via src/ path added by db_helper) ✓
        └→ utils/logger ✗ FAILED

Root Cause:
- db_helper.py adds src/ to sys.path
- database.py is imported from src/storage/database.py
- database.py tries to add root path AFTER being imported
- But by then, the import context is already in src/

================================================================================
                            THE SOLUTION
================================================================================

Use try/except approach - try importing first, if fails then add path:

```python
# src/storage/database.py
try:
    from utils.logger import setup_logger
except ImportError:
    # Only add path if import fails
    current_dir = os.path.dirname(os.path.abspath(__file__))
    root_dir = os.path.dirname(os.path.dirname(current_dir))
    if root_dir not in sys.path:
        sys.path.insert(0, root_dir)
    from utils.logger import setup_logger
```

Why this works:
1. If database.py is imported normally (from src/), it tries direct import
2. If that fails, it calculates and adds root path
3. Works in both contexts: direct import AND via db_helper
